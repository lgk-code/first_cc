# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目简介

这是一组**自包含的单文件浏览器小玩具**。每个 `.html` 文件都是一个独立、完整的应用——HTML 结构、CSS 样式、JS 逻辑全部内联在同一个文件里，彼此之间没有共享代码或依赖。没有构建系统、包管理器、依赖、测试套件或 lint 配置；不需要编译或转译。

当前包含：

- **`snake.html`** —— 单色 LCD 复古风的贪吃蛇游戏。
- **`pixel.html`** —— 干净现代浅色界面的像素画板（在网格上涂色、导出 PNG）。

新增玩具时沿用同样的形态：一个新的独立 `.html` 文件，零依赖。

## 运行方式

直接用浏览器打开对应文件即可，没有服务器或构建步骤：

```sh
start snake.html      # Windows
start pixel.html
```

修改后刷新浏览器（用 Ctrl+F5 强制刷新以绕过缓存）。除此之外不需要运行任何东西。

## 架构（`snake.html` 内部）

所有逻辑都在单个 `<script>` 块里。下面这几点是"读代码不容易一眼看出来"、动手前值得先了解的设计：

- **渲染与逻辑解耦。** 渲染通过 `requestAnimationFrame`（`frame`）每帧执行，但游戏*逻辑*按固定步长推进：只有当 `gameClock` 自上一步以来累计满 `speed` 毫秒，`step()` 才会触发。`gameClock` 仅在非暂停/非死亡时累加真实时间差，所以暂停能干净地冻结移动和金苹果计时。
- **插值移动。** 蛇在网格上以格子（而非像素）存储逻辑位置。为了渲染格子间的平滑滑动，`draw(t)` 用 `t`（距下一步的进度比例）在每段的上一位置（`lastSnake` / `segFrom`）和当前位置之间插值。`grew` 记录上一步是否新增了身体段，它决定 `segFrom` 如何把旧位置映射到新位置。
- **坐标系。** 所有逻辑量以格子为单位（`COLS` × `ROWS`）；乘以 `CELL` 得到像素。画布尺寸由这些推导得出。
- **穿墙模式。** `wrapMode` 开启时，蛇头位置对网格取模。由于穿墙的段会瞬移到棋盘另一侧，渲染时会检测跨度 > 1 格的跳变并对该段跳过插值（否则会在屏幕上拉出一条可见的横线）。任何新增的移动/拖影特效都必须套用同样的跳变守卫。
- **金苹果。** 一个限时金苹果（分值 `BONUS_SCORE`）可能和普通食物一起刷新；它在 `BONUS_TTL` 后过期（以 `gameClock` 计时），并渲染一圈逐渐缩短的环形计时器。
- **单色 LCD 渲染。** 画面被刻意做成一块单色橄榄绿 LCD。`drawMatrix()` 每帧填充背光底色、画点阵纹理，并调用 `drawBorder()` 画场地边界——常规模式是实心墙框、穿墙模式是虚线框，**边框样式本身用来表达当前模式**。蛇和食物是深墨像素；唯一的彩色（琥珀 `AMBER`）只保留给金苹果，作为视觉记忆点。背景与边框都画在 canvas 内部（而非用 CSS），这样 canvas 被响应式缩放时点阵和边框仍与格子精确对齐。
- **分模式最高分。** 每个模式各记各的最高分：`snakeBestNormal`（常规）与 `snakeBestWrap`（穿墙），存于 `localStorage`。`loadBest()` 在启动和切换模式时按当前 `wrapMode` 读取对应键；旧版共用的 `snakeBest` 键会被迁移归入常规模式。
- **外观为纯 CSS。** 掌机机身、屏框、通电闪屏动效、实体按键等全部由 CSS 实现，与游戏逻辑无关；改视觉时优先动 CSS。动效在 `prefers-reduced-motion` 下会关闭。

## 架构（`pixel.html` 内部）

同样是单个 `<script>` 块。值得先了解的设计：

- **数据 → 重绘的单向流。** 真值就是二维数组 `grid[r][c]`（每格存颜色字符串，空格为 `EMPTY = null`）。所有交互（涂色、橡皮、清空、撤销）都只改 `grid`，然后调 `draw()` 全量重绘整张 canvas。网格只有 16×16 或 32×32，全量重绘足够便宜，不需要做局部刷新。
- **尺寸由 `BOARD` 推导。** 画布显示边长固定为 `BOARD`（像素），`CELL = BOARD / COLS`。切换网格大小时只改 `COLS/ROWS`、重算 `CELL`、重建 canvas，显示尺寸保持不变。
- **按 DPR 放大保清晰。** `setupCanvas()` 把 canvas 的内部像素尺寸按 `devicePixelRatio` 放大、再用 CSS 缩回显示尺寸，并 `setTransform` 重置后重新 `scale(dpr,dpr)`——**切换网格大小时必须重置变换**，否则缩放会累积。
- **分尺寸独立存档。** 每种网格大小各存各的：`localStorage` 键形如 `pixelart-16` / `pixelart-32`，另有 `pixelart-size` 记住上次用的尺寸。`loadGrid()` 读取时**校验维度**（行列数必须等于目标尺寸），无存档或数据损坏/旧格式则回退到该尺寸的**默认画作**。
- **默认画作 / 重置。** 每种尺寸有一幅内置默认图（`ART_16` 红蘑菇、`ART_32` 热气球），用**字符画**描述：每行一个字符串，字符经 `ART_LEGEND` 映射成颜色，空格 = 空格子；`artToGrid()` 按目标尺寸裁剪/补齐（容忍行长不齐）。默认图既是首次打开的初始画面（经 `loadGrid` 回退），也是「重置」按钮的目标（`defaultGrid(COLS)` → `draw()` → `commit()`）。改默认图就编辑这两个字符画数组即可，**字符要落在 `ART_LEGEND` 里，否则当作空格**。
- **"一笔"是历史与存档的提交单位。** 一次按住拖动会连续涂多格，但只在 `pointerup` 调一次 `commit()`——`commit()` 同时负责存档（`saveState()`）和压入撤销历史。因此撤销是**整笔一起回退**，不是一格一格退。新增任何会改 `grid` 的操作都应在操作完成后调用 `commit()`，否则不会进历史、也不会自动保存。
- **撤销/重做。** `history` 是一串 `grid` 深拷贝快照，`histIndex` 指向当前步。撤销/重做即移动指针并恢复对应快照；撤销后又有新操作会**砍掉重做分支**。历史上限 `MAX_HISTORY`。切换网格大小会 `resetHistory()`，即每种尺寸的撤销历史相互独立。快照用 `snapshot()`（逐行 `slice`）深拷贝，恢复时也要再拷贝，避免后续涂改污染历史里的旧快照。
- **导出。** 导出时**另建一张离屏 canvas**（每格放大到固定像素）只填已涂的格子，空格保持透明——所以导出的 PNG 适合做透明背景的像素图标，而非把屏幕上的网格线一起截下来。
